## 1. 產品定位

本系統針對四種不同投資風格的使用者，各自提供一套獨立的 AI 評分邏輯。每套邏輯會為股票產生統一的三色燈號（🟢🟡🔴），並在前台以「人話版理由」呈現，不顯示原始數字或公式。

**設計原則：**
- 燈號結果小白也能看懂，3 秒內掌握重點
- 數字與公式僅保留於後台日誌
- 各族群邏輯相互獨立，同一檔股票在不同族群可有不同燈號
- 手機上同樣好用

## 2. 族群一：穩健存股族

### 2.1 適用使用者

長期持有、不常盯盤、以配息為主要收益來源的保守型投資人。

### 2.2 免費資料來源

| 資料 | 來源 |
|------|------|
| 個股每日殖利率、PBR | 證交所每日公布（TaiwanStockPER） |
| 歷季 EPS | 公開資訊觀測站季報（TaiwanStockFinancialStatements） |
| 歷年配息紀錄 | 公開資訊觀測站（TaiwanStockDividend） |

### 2.3 後台 AI 運算邏輯

#### Step 1 — 穩定性前置過濾（最優先）

在進行任何估值判斷前，先確認是否為真正的穩健存股。**三項條件任一不符，直接標記「不適合存股」，不發出任何燈號。**

| 條件 | 門檻 | 說明 |
|------|------|------|
| 配息連續性 | 近 5 年配息 ≥ 4 年 | 排除不穩定或暫停配息的股票 |
| 盈利波動 | 近 8 季 EPS 標準差 / 均值 < 50% | 排除景氣循環型高波動股 |
| 盈利方向 | 近 4 季 EPS 合計 > 0 | 排除目前虧損股 |

> **設計意圖：** 航運（長榮、陽明）、鋼鐵（中鋼）等景氣循環股雖然有年配息，但 EPS 波動極大，不適合視為穩健存股，應在此階段過濾。台積電等成長型龍頭也可能因 EPS 波動不符門檻，前台標示改為「建議參考成長族評分」而非直接說「不適合存股」，避免與使用者認知衝突。

#### Step 2 — 存股安全等級標記

| 等級 | 條件 | 典型族群 |
|------|------|---------|
| A 級 | 近 5 年全數配息 + EPS 變異係數 < 20% | 金融、電信、穩定龍頭 |
| B 級 | 配息 ≥ 4/5 年 + EPS 變異係數 20–50% | 景氣循環但相對穩定 |
| C 級 | 勉強通過過濾，波動偏高 | 顯示警示，謹慎參考 |

#### Step 3 — Graham 內在價值計算

$$\text{合理價} = \sqrt{22.5 \times \text{近 4 季 EPS} \times \text{每股淨值}}$$

**防護條件：** 若近 4 季 EPS ≤ 0 或每股淨值 ≤ 0，標記「無法估值」，不執行公式。

每股淨值計算：$\text{每股淨值} = \text{當前股價} \div \text{PBR}$

#### Step 4 — 殖利率歷史區間判定

拉出近 2 年日殖利率資料，計算歷史均值（μ）與標準差（σ）。

| 燈號 | 條件 | 說明 |
|------|------|------|
| 🟢 可布局 | 殖利率 ≥ 5%，或殖利率 > μ + 1σ | 股價已跌出歷史甜蜜點 |
| 🟡 觀察等待 | 殖利率落在 μ ± 1σ 以內 | 目前在歷史均值附近，合理但非甜蜜點 |
| 🔴 偏危險 | 殖利率 < μ − 1σ，或股價 > Graham 合理價 × 1.3 | 股價過熱或嚴重高估 |

## 3. 族群二：產業成長族

### 3.1 適用使用者

關注企業成長動能，願意承受較高估值，以資本利得為主要目標的中長線投資人。

### 3.2 免費資料來源

| 資料 | 來源 |
|------|------|
| 每月營收 | 公開資訊觀測站每月 10 號前（TaiwanStockMonthRevenue） |
| 季報 EPS、毛利率 | 公開資訊觀測站（TaiwanStockFinancialStatements） |
| 每日 PE 數據 | 證交所（TaiwanStockPER） |
| 法人買賣明細 | 證交所（TaiwanStockInstitutionalInvestorsBuySell） |

### 3.3 後台 AI 運算邏輯

#### 多因子評分系統（100 分制）

| 評分項目 | 權重 | 計算方式 | 細則 |
|---------|------|---------|------|
| 營收成長力道 | 35 分 | 近 12 個月累積營收 YoY | ≥30%→35分、≥15%→25分、≥5%→15分、負成長→0分；cap 上限 80% |
| 毛利率趨勢 | 20 分 | 最新季 vs 前一季毛利率差（百分點） | 擴張≥+2%→20分、持平≥0%→14分、小幅收縮≥-3%→8分、大幅收縮→0分 |
| EPS 成長 | 20 分 | 近 4 季 EPS 合計 vs 前 4 季 EPS 合計 | ≥30%→20分、≥10%→14分、≥0%→8分、衰退→0分 |
| 估值合理性 | 15 分 | 當前 PE 落在個股近 2 年 PE 歷史 percentile | ≤30th→15分、≤50th→11分、≤70th→6分、>70th→0分 |
| 法人動向 | 10 分 | 投信 50% + 外資 30% + 自營商 20%，近 15 日合計 | 淨買超→10分、淨賣超→0分；天數待觀察，若訊號落後明顯可調整至 20 日 |

> **估值採 PE percentile 的原因：** 不同市場環境下合理 PE 差異極大。台積電歷史 PE 區間約 12–28，當前 PE 若在第 90 百分位即代表在其個股歷史上屬貴，直接扣分，不受外部市場環境影響。

#### 三段燈號

| 燈號 | 條件 |
|------|------|
| 🟢 強勢成長，可布局 | 總分 ≥ 85，**且** PE 百分位 ≤ 80th |
| 🟡 成長持續，觀察 | 總分 70–84；或總分 ≥ 85 但 PE 百分位 > 80th |
| 🔴 成長趨緩或估值過熱 | 總分 < 70 |

> **估值硬封頂規則：** PE 百分位 > 80th 時，無論總分多高，最高燈號為 🟡，並標注「估值偏高，不建議現在介入」。多頭市場全面高估值時，GREEN 訊號自然收斂，不需人工干預。

## 4. 族群三：趨勢波段族

### 4.1 適用使用者

以波段操作為主，持股 2–4 週，追蹤趨勢啟動訊號的中線投資人。

### 4.2 免費資料來源

| 資料 | 來源 |
|------|------|
| 每日收盤行情 | 證交所（TaiwanStockPrice） |
| 三大法人買賣明細 | 證交所（TaiwanStockInstitutionalInvestorsBuySell） |
| 個股流通在外股數 | 公開資訊觀測站 |

### 4.3 後台 AI 運算邏輯

#### 均線扣抵確認

| 條件 | 說明 |
|------|------|
| MA20 扣抵通過 | 今日收盤 > 20 個交易日前收盤（月線扣抵轉正） |
| MA60 扣抵通過 | 今日收盤 > 60 個交易日前收盤（季線扣抵轉正） |

兩項皆通過，代表月線與季線均處於上彎趨勢，是波段多頭的基本條件。

#### 法人強度分數（動態合成）

$$\text{法人強度分數} = 0.5 \times \text{投信強度} + 0.3 \times \text{外資強度} + 0.2 \times \text{自營商強度}$$

各強度 = 近 3 日該法人淨買超張數（正為買、負為賣）

> **Phase 2 升級項目：** 主力券商集中度（規劃中 20% 權重）需要 `TaiwanStockTradingDailyReport`，FinMind 免費方案不支援。Phase 1 以自營商替代；升級付費帳號後調整為：投信 40% + 外資 30% + 主力券商 20% + 自營商 10%。

**投信控盤率動態門檻（依市值規模調整）：**

| 股票規模 | 市值參考 | 觸發門檻 |
|---------|---------|---------|
| 小型股 | < 100 億 | 控盤率 > 0.5% |
| 中型股 | 100–1,000 億 | 控盤率 > 0.1% |
| 大型股 | > 1,000 億 | 控盤率 > 0.03% |

> 分母統一使用流通在外股數（已扣除庫藏股），避免庫藏股造成控盤率虛高。

#### 三段燈號

| 燈號 | 條件 |
|------|------|
| 🟢 多方動能，可布局 | MA20 + MA60 扣抵均通過，且法人強度分數 > 門檻，**且今日成交量 ≥ 20 日均量 × 1.2** |
| 🟡 趨勢在但動能待確認 | 扣抵值通過但量未達標（< 20日均量 × 1.2），或法人強度不足 |
| 🔴 跌破關鍵均線 | 股價跌破 MA20 且最新股價 < 扣抵值（均線即將下彎） |

> **量能門檻說明：** 1.2 倍為保守門檻（相較短線族的 2 倍），目的是過濾無人交易的假突破，不要求爆量，只確認有足夠市場參與。

**建議出場：** 停損跌破 MA20 / 目標報酬 +15% / 最長持有 20 個交易日

## 5. 族群四：短線動能族

### 5.1 適用使用者

以隔日或 3–5 日為操作週期，追蹤爆量強勢訊號、需要快速進出的短線交易者。

### 5.2 免費資料來源

| 資料 | 來源 |
|------|------|
| 每日收盤行情、成交量 | 證交所（TaiwanStockPrice） |
| 三大法人買賣明細 | 證交所（TaiwanStockInstitutionalInvestorsBuySell） |
| 盤中即時成交量 | 證交所盤中資料（需確認免費方案權限） |
| 券商分點明細（Phase 2） | 證交所 TaiwanStockTradingDailyReport（需付費） |

### 5.3 後台 AI 運算邏輯

#### Phase 1：盤後收盤資料版（免費可用）

使用收盤後資料，隔日開盤前推播訊號：

**觸發條件（同時成立）：**
- 今日成交量 > 5 日均量 × 2（爆量）
- 今日收盤漲幅 ≥ 3%
- 收盤位置 ≥ 60%（收盤站在今日高低點區間的上 40%，確保是紮實多方 K 棒，排除長上影線與收中段的弱勢爆量）

#### Phase 1 盤中版（中午 12:30，需確認權限）

$$\text{修正預估量} = \frac{\text{12:30 累積量}}{210} \times 270 \times 1.15$$

係數 1.15 說明：台股尾盤最後 60 分鐘成交量平均佔全日 28–32%，高於線性預測的 22%，保守取 1.15。

#### Phase 2：雙層隔日沖過濾器（需付費資料）

> **⚠️ 需要 `TaiwanStockTradingDailyReport`，FinMind 免費方案不支援。升級後啟用。**

**第一層 — 即時行為偵測（當日 12:30）：**
識別昨日大量買超、今日已開始出貨的分點，觸發條件：已出貨佔昨日買超 > 30%。

**第二層 — 歷史持股習慣過濾：**
抓取昨日買超前幾名分點的近 20 日平均持股天數，< 2 天者標記為高風險隔日沖分點。

> 雙層設計解決「主力下午才倒貨、12:30 看不到」的實務漏洞。

#### 三段燈號

| 燈號 | Phase 1 條件 | Phase 2 條件（加強版） |
|------|-------------|----------------------|
| ⚡🟢 安全動能 | 爆量 + 漲幅 ≥ 3% + 收盤位置佳 | 同左 + 雙層過濾均無警示 |
| ⚡🟡 有量需謹慎 | 量能達標但收盤位置偏低 | 量能達標但第二層歷史風險偏高 |
| ⚡🔴 動能不足或警示 | 量能未達標，或收盤收黑 | 任一層觸發高風險 |

**建議出場：** 停損 -4% / 目標報酬 +6% / 最長持有 5 個交易日

## 6. 市場情境提示

顯示於 Dashboard 頂部，作為輔助判斷依據，不納入個股評分公式。

### 6.1 顯示內容

| 項目 | 資料來源 | 說明 |
|------|---------|------|
| 加權指數 | 證交所 | 今日收盤 / 漲跌幅 |
| 台指期夜盤 | 期交所 / 外部 API | 昨夜期貨漲跌，反映隔夜外資預期 |
| 費城半導體指數 | 外部 API（Yahoo Finance 等） | 台股科技股重要領先指標 |
| 美股那斯達克 | 外部 API | 整體科技股情緒 |

### 6.2 顯示邏輯

- 夜盤漲跌 > +1%：顯示綠色上箭頭，文字「海外市場偏強」
- 夜盤漲跌 < -1%：顯示紅色下箭頭，文字「海外市場偏弱，訊號需更謹慎」
- 介於 ±1%：顯示灰色，文字「海外市場平盤」

### 6.3 消息面

突發消息（法說會、合約、政策）不納入自動評分。個股頁提供「查詢公開資訊觀測站」快捷連結，讓使用者自行確認近期重大公告。Phase 2 可考慮整合結構化公告（月營收超預期、EPS 驚喜）作為額外加分項目。

## 7. 前台顯示原則

### 7.1 只顯示人話版理由

所有數字（PE percentile、控盤率、標準差、預估量係數）**僅保留在後台**，不對使用者呈現。

**存股族卡片：**
```
中華電  🟢 A級 可布局

✅ 近 5 年連續配息，獲利穩定
✅ 目前殖利率高於歷史平均
✅ 股價接近 Graham 合理價
```

**存股族卡片（未通過過濾，成長型龍頭）：**
```
台積電  📊 穩健存股條件不符

⚠️ EPS 波動高於穩健存股門檻
→ 建議參考「成長族」評分
```

**成長族卡片（估值過熱版）：**
```
台積電  🟡 成長持續，估值偏高

✅ 營收、EPS 持續成長
⚠️ 目前股價處於近兩年估值高點
   建議等待回檔或更好的買點
```

**趨勢波段族卡片：**
```
廣達  🟢 多方動能，可布局

✅ 月線、季線均呈上升趨勢
✅ 法人持續買進，動能明確
📌 建議停損：跌破月均線
```

**短線動能族卡片：**
```
緯創  ⚡🟢 安全動能

✅ 今日成交量為近期均量 3 倍以上
✅ 漲幅強勁，收盤站穩高點
📌 建議停損：-4%  目標：+6%
```

### 7.2 統一燈號格式

| 族群 | 燈號 |
|------|------|
| 存股族 | 🟢 可布局 / 🟡 觀察等待 / 🔴 偏危險 |
| 成長族 | 🟢 強勢成長 / 🟡 觀察 / 🔴 過熱或趨緩 |
| 趨勢波段族 | 🟢 多方動能 / 🟡 趨勢在，待確認 / 🔴 跌破均線 |
| 短線動能族 | ⚡🟢 安全動能 / ⚡🟡 需謹慎 / ⚡🔴 不建議 |

## 13. 自選追蹤清單 & 歷史快照

> **版本：** v1.3 新增
> **目標：** 讓使用者把感興趣的個股加入自選清單，每日自動記錄燈號與評分，事後可回頭驗證 AI 推薦是否準確。

---

### 13.1 自選追蹤清單 Tab

在主頁 Tab 列新增第 5 個 Tab「⭐ 自選清單」，位置排在四大族群之後。

**Tab 進入頁面內容：**

```
⭐ 自選清單                              + 新增股票

┌─────────────────────────────────────────────────────┐
│  台積電  2330                                        │
│  成長族 🟢 90分  趨勢族 🟡 65分  今日 +1.2%  ▸      │
│  加入時燈號：成長族 🟢 85分  (2026-05-20)            │
│  持有 2 天 · 成長族報酬 +3.4%                        │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│  廣達  2382                                          │
│  短線族 ⚡🟡 67分  趨勢族 🟢 90分  今日 +4.1%  ▸     │
│  加入時燈號：短線族 ⚡🟢 100分  (2026-05-21)          │
│  持有 1 天 · 短線族報酬 +4.1%                        │
└─────────────────────────────────────────────────────┘
```

**欄位說明：**

| 欄位 | 說明 |
|------|------|
| 今日各族群燈號 | 即時從快取取得，顯示最近一次計算結果 |
| 今日漲跌 | 最新收盤 vs 前一日收盤 |
| 加入時燈號 | 使用者點擊「加入追蹤」當下記錄的族群/分數/日期 |
| 持有天數 | 從加入日起算的交易日數 |
| 報酬率 | (當前收盤 − 加入當日收盤) / 加入當日收盤 × 100% |

---

### 13.2 新增股票流程

1. **方式一（推薦）：** 在各族群列表的股票卡片右側，點擊「⭐ 追蹤」按鈕 → 自動加入
2. **方式二：** 點「+ 新增股票」→ 輸入代號或名稱搜尋 → 確認加入

加入時自動記錄快照：
- 加入日期
- 各族群當下燈號與分數（成長/趨勢/短線/存股）
- 當日收盤價（作為基準價格）

---

### 13.3 燈號歷史時間軸

點擊自選清單中任一股票 → 展開歷史燈號時間軸：

```
台積電 2330  成長族燈號變化

  05/20  🟢 90分  ←── 加入當天
  05/21  🟢 88分
  05/22  🟡 72分  ⚠️ 分數下滑
```

顯示規則：
- 每個有資料的交易日顯示一格
- 燈號改變時（綠→黃、黃→紅）加上 ⚠️ 提示
- 最多顯示近 30 個交易日

### 13.4 準確度統計面板

在自選清單頂部顯示一個小統計面板：

```
我的追蹤績效（過去 30 日）

成長族綠燈後 10 日均報酬：  +4.2%   12 筆
趨勢族綠燈後 20 日均報酬：  +6.8%    8 筆
短線族綠燈後  5 日均報酬：  +1.9%    5 筆
```

計算邏輯：
- 取使用者追蹤過的股票中「加入時為綠燈」的紀錄
- 計算加入後第 5 / 10 / 20 個交易日的報酬率
- 樣本數 < 5 筆時不顯示，避免資料不足誤導

### 13.5 每日歷史快照（後端）

**儲存方式：Node 24 內建 `node:sqlite`（零外部依賴）**

資料表結構：

```sql
-- 每日族群計算結果快照
CREATE TABLE IF NOT EXISTS daily_signals (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  date        TEXT NOT NULL,        -- 'YYYY-MM-DD'
  tribe       TEXT NOT NULL,        -- 'growth' | 'trend' | 'momentum' | 'dividend'
  stock_id    TEXT NOT NULL,
  stock_name  TEXT NOT NULL,
  score       INTEGER NOT NULL,
  signal      TEXT NOT NULL,        -- 'green' | 'yellow' | 'red' | 'gray'
  tag         TEXT,
  close_price REAL,
  created_at  TEXT DEFAULT (datetime('now','localtime')),
  UNIQUE(date, tribe, stock_id)
);

-- 使用者自選清單
CREATE TABLE IF NOT EXISTS watchlist (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  stock_id     TEXT NOT NULL UNIQUE,
  stock_name   TEXT NOT NULL,
  added_date   TEXT NOT NULL,       -- 'YYYY-MM-DD'
  added_price  REAL,                -- 加入當日收盤價
  added_signals TEXT,               -- JSON：加入當下各族群燈號快照
  note         TEXT,                -- 使用者備註（選填）
  created_at   TEXT DEFAULT (datetime('now','localtime'))
);
```

**觸發時機：**
- 每次 `/api/stocks?tribe=X` 計算完成後，自動將結果寫入 `daily_signals`
- 同一天同族群重複計算時，以最新結果覆蓋（UPSERT）

**新增 API 端點：**

| 端點 | 說明 |
|------|------|
| `GET /api/watchlist` | 取得自選清單（含今日燈號） |
| `POST /api/watchlist` | 新增股票 `{ stock_id, stock_name }` |
| `DELETE /api/watchlist/:id` | 移除股票 |
| `GET /api/history/:stock_id` | 取得個股各族群歷史燈號（近 30 日） |
| `GET /api/accuracy` | 計算追蹤績效統計 |
